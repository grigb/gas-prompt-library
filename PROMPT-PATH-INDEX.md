## Closeout Routing Note

Routine owner-facing and agent-facing session closeout uses `CREATE-SESSION-RECORD` at `~/.agents/prompts/creation/CREATE-SESSION-RECORD.md`.
`CLOSE-STEWARD` remains discoverable only as an internal Project Steward preflight/debug reference invoked through the unified session-record flow; do not route owners to it directly for routine closeout.
`CLOSE-SUPERVISOR` remains discoverable only as an internal Blocker Supervisor preflight/debug reference invoked through the unified session-record flow; do not route owners to it directly for routine closeout.

## Usage Instructions for AI Agents

### STEP 1: Check This Index FIRST
Before searching, read this file to find the exact path.

### STEP 2: Direct Read
Use the exact path from the table above.

### STEP 3: Only Search If Not Listed
If the file isn't in this index, THEN use search tools.

## Common Patterns

- **Creation tasks**: Check `creation/` directory
- **Session closeout**: Use `creation/CREATE-SESSION-RECORD.md`; do not use `CREATE-AUDITABLE-RECORD` + `HANDOFF`, `CLOSE-STEWARD`, or `CLOSE-SUPERVISOR` directly for routine closeout
- **Work orders**: Check `work-orders/` directory
- **Handoffs**: Check `handoffs/` directory
- **Agent status relay**: Use `triage/agent-status-update-for-routing.md`
- **Specialized agents**: Check `agents/` directory
- **Modes**: Check `modes/` directory
- **General prompts**: Check `general/` directory
- **Project-specific K2B directives**: Check `handoffs/k2b-project-directives/` (never `general/`)

## Path Resolution

All paths use `~/.agents/prompts/` as the base directory.

Example resolution:
- Index shows: `~/.agents/prompts/creation/CREATE-WORK-PROPOSAL.md`
- Full path: `~/.agents/prompts/creation/CREATE-WORK-PROPOSAL.md`
